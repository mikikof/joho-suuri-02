# new-lecture — 情報数理入門の新しい回を段階的に scaffold

このプロジェクトは「情報数理入門」の各回を `lecNN/` 配下に **html + xlsx + (要求時) pptx** で管理している。新しい回を追加する時はこの skill を使って、第2回 (`lec02/`) と同じ品質・スタイルで作る。

---

## 動作原則 — 必ず守ること

1. **段階的に進める** — HTML → Excel → (要求があれば) PPTX の順で 1 段階ずつ完成させる。複数同時に作らない
2. **各段階の完了後に必ずユーザー確認** — 次の段階へ進む前に「OKか / 修正点があるか」を聞く。承認なしに次へ進まない
3. **テンプレートは lec02** — 構造・配色・命名規約・PART 構成パターンは lec02 を必ず先に読み込んでから真似る。新しいスタイルを発明しない
4. **HTML が真実の源 (source of truth)** — Excel と PPTX は HTML の内容にタイアップさせる。HTML の PART 構成・お題・数式と矛盾させない
5. **PPTX は要求があった時だけ作る** — ユーザーから明示的に頼まれない限り着手しない

---

## 入力収集 (最初に対話で確認)

- **回数 N** (整数、2桁ゼロ詰めで `lec0N` を作る)
- **テーマ** (例: 「確率と統計の基礎」「微分入門」)
- **参考資料**:
  - 参考 URL があれば WebFetch / WebSearch で内容を取り込む
  - 既存の画像 / PDF / pptx があれば Read で取り込む
  - 教科書・シラバス等の指定があれば確認
- **PART 構成** (章立て): 5 章前後。提案して承認をもらう
- **各 PART の身近なお題**: ユーザーから引き出す or 提案

入力が揃ったら **PART 構成案 (各 PART のタイトル / お題 / グラフで何を見せるか)** を箇条書きで提示し、承認をもらってから先へ進む。

### PART 構成パターン (lec03 で確立した骨格)

新しい回でも、まずこの骨格を提案案として出す:

1. **イントロ 3 枚** — 表紙 / 全体像 / 問いから始めるティーザー
2. **PART 1 (概念の入口)** — メインの概念を「身の回り → 金融への展開」の 2 ステップで導入
   - **1-1**: 教室の座席・地図・スマホ画面など、生徒が直感で分かる例で概念を紹介
   - **1-2**: 同じ発想を金融題材（リスクリターン散布図、投資判断 等）に転用して見せる
3. **PART 2〜4** — 概念を発展させる中段。各 PART の身近なお題は金融軸（ATM、配達エリア、家計、料金プラン 等）と物理軸（地図、Wi-Fi 範囲、当たり判定 等）を混ぜて提示
4. **PART 5 (発展・締め)** — AI / データサイエンスへの接続。「同じ式が世界中で動いている」を体感させる **リッチなインタラクティブを 2-3 スライドぶん** 用意:
   - 例: 映画レコメンドの散布図 / Spotify 風カードグリッド（リアルタイム並び替え + マッチ度バッジ）/ SVG パラメトリック顔の認証シミュレーション
   - 座標にこだわりすぎず、シミュレーション的な「リッチさとインパクト」を優先
5. **SUMMARY** — 3 つの統一原理で締め

**NEXT スライド（次回予告）は作らない**。次回の内容を縛るとフローが固くなる。

### 「身近 → 金融」の繋ぎ方（題材選びのヒント）

- 各 PART の入り口は「身近な現象」、ただし**金融題材を必ず並走させる**
- 金融ネタは概念の汎用性を示す格好の材料（ATM 検索 = 領域判定、リスクリターン = 散布図、投資ポートフォリオ = 距離計算 等）
- ユーザーは金融ネタを優先する傾向。提案時に「身近な例 + 金融題材」をセットで出す

## 生成ステップ — この順番で進めること

### 事前準備 (毎回)

- `lec02/index.html` を読んで規約を確認 (`.claude/skills/new-lecture/TEMPLATE_GUIDE.md` も参照)
- `lec0N/` ディレクトリを作成: `mkdir -p lec0N`

---

### Phase 1: HTML (対話型 web スライド) — 最初に作る

**目的**: 講義の本体。スライダー + Chart.js でテーマを体感させる。これが Excel / PPTX のベースになる。

**手順**:

1. `lec02/index.html` をコピーして開始:
   ```bash
   cp lec02/index.html lec0N/index.html
   ```
2. `TEMPLATE_GUIDE.md` の規約に従って書き換える:
   - `<title>` を「情報数理入門 第N回 — {テーマ}」に
   - サイドバーのナビ項目とアンカー (`#part1` 等) を新 PART に合わせる
   - 各 PART を「身近なお題 → スライダー → 数式 → グラフ → 解釈」の流れで書き換え
   - ID プレフィックス `cN-*` を維持 (PART 1 → `c1-*`, PART 2 → `c2-*`)
   - IIFE は PART 単位で独立させる
   - 必要に応じて WebFetch で参考 URL から数式 / 例題を取り込む
3. ブラウザで `lec0N/index.html` を開いて動作確認するようユーザーに案内
4. **ここで一度停止** — 「HTML はこれで OK か / 修正点はあるか」をユーザーに聞く

**承認が出るまで Phase 2 には進まない**。修正があれば反映してから再度確認。

---

### Phase 2: Excel 演習 — HTML が確定してから着手

**目的**: HTML で学んだ内容を手を動かして定着させる。HTML より **少しレベルアップした演習** を含む。

**ターゲット環境は Google スプレッドシート** (Excel ではない)。詳細規約は `EXCEL_GUIDE.md` を参照。要点だけ:

- 黄色のアクション枠で操作指示（「黄色を埋める」「グラフで可視化」「動かしてみる」の 3 種類）
- 「問い」「予想」「ヒント」「発展」セクションは作らない（ユーザーは演習を「表を関数で埋める → グラフで可視化する」のシンプルな型に揃えたい）
- 散布図・線描画・軸固定・系列追加・フィルタなど、操作手順は Sheets の UI 用語で書く
- 設問の数値は「半径 5 で 7 件」のように**答えがフィットする**ように調整する

**手順**:

1. ユーザーに開始の合図をもらう (Phase 1 承認時に「Excel に進んで OK」と確認)
2. python3 + `openpyxl` で生成。シート構成は以下に統一:

   ```
   PART 別シート × N 個   (HTML の各 PART と 1 対 1 対応、内容も合わせる)
   演習① 基本             (HTML より一段レベルアップ、表を埋めて可視化)
   演習② 応用             (多次元・複合条件、表を埋めて可視化)
   挑戦① 〜               (発展題材、表を埋めて可視化)
   挑戦② 〜               (試行錯誤型、入力セルを動かして最適化)
   ```

3. **HTML の内容と必ずタイアップさせる**:
   - PART 別シートのタイトル / お題 / 例題は HTML と同じ
   - 演習 / 挑戦は HTML のお題を素材にしつつ、変数や条件を変えてレベルアップ
4. 各シートのレイアウトと黄色アクション枠の使い方は `EXCEL_GUIDE.md` に従う
5. **数式の検算**: 生成後に LibreOffice で convert して `data_only=True` で評価値を確認:
   ```bash
   soffice --headless --calc --convert-to xlsx --outdir /tmp/eval <生成ファイル>
   python3 -c "import openpyxl; wb = openpyxl.load_workbook('/tmp/eval/...', data_only=True); ..."
   ```
6. **ここで一度停止** — 「Excel はこれで OK か / 修正点はあるか / PPTX も作るか」をユーザーに聞く

---

### Phase 3: PPTX — ユーザーから明示的に要求された時だけ

**目的**: 配布 / 印刷 / 対面授業用の PowerPoint。HTML と Excel の内容をスライド形式に再構成。

**起動条件**: ユーザーが「pptx も作って」「PowerPoint を作成して」など明示的に頼んだ時のみ。聞かれてないのに作らない。

**手順**:

1. python3 + `python-pptx` で生成。lec02 の元 PDF 構成に倣う:
   - 23 スライド前後（lec03 と同じく **NEXT は作らない**）
   - 表紙 → 目次 → 各 PART の章扉 + 本文 + 演習 → まとめ
   - 配色: メイン青 `#2563EB`、深青 `#1E3A8A`、文字 `#1F2937`
   - 和文フォント: Yu Gothic / Hiragino Sans / Zen Kaku Gothic New
2. **HTML / Excel と内容を一致させる**:
   - PPTX の章立て = HTML の PART 構成
   - 演習スライドは Excel の演習問題と同じ題材
3. 画像が提供されていれば対応するスライドに埋め込む
4. スクリプトは Bash heredoc でその場で書いて実行 (ファイルとして保存しない)

---

## 完了処理 (各 Phase で個別に or 最後にまとめて)

- ルート `index.html` の目次更新:
  - `<div class="placeholder">` の `0N` 部分を `<a class="lec-card" href="lec0N/index.html">...</a>` に置換
  - 説明文に PART のキーワードを並べる (lec02 のカードと同じ粒度)
  - 次の回 `0(N+1)` の placeholder を新規追加
- `git status` で変更確認
- コミット粒度はユーザーに確認 (Phase ごとに 1 コミット / 全部まとめて 1 コミット どちらでも)
- プッシュは聞いてから

---

## 注意

- **lec02 / lec03 を壊さない**: 既存ファイルは読むだけ
- **CDN/フォントの URL は変えない**: lec02 と同じ Chart.js 4.4.1 / Google Fonts を使う
- **日本語のファイル名**: `情報数理入門_第N回.pptx` のように半角アンダースコア + 全角「第N回」で統一
- **ID 衝突**: PART N の DOM ID は `cN-*` プレフィックス必須 (詳細は TEMPLATE_GUIDE.md)
- **生成スクリプトは保存しない**: pptx/xlsx 生成用の python スクリプトは Bash で一回実行するだけ。ファイルとしてコミットしない
- **段階の飛び越し禁止**: HTML 未完成のまま Excel に着手したり、Excel 未完成のまま PPTX に着手したりしない
